2. Threading Model

Three concurrent components share data through thread-safe structures.

Kafka Consumer Thread
Polls market-data.ticks
Feeds ticks to FeatureEngine
daemon thread
gRPC Thread Pool (10)
Serves GetSignal RPCs
Reads features from engine
managed by grpcio
FastAPI Main Thread
/health, /ready, /metrics
/v1/models/reload
uvicorn event loop
Thread Safety: FeatureEngine uses threading.Lock — the consumer thread writes bar data and features, while gRPC threads read them. The lock ensures no thread sees a half-updated feature vector. SignalModel has its own separate lock for atomic model hot-reload.
